# Расширенный технический отчет по экосистеме `tests_vinkom` (Март 2026)

Этот документ представляет собой углубленный анализ всех 32 файлов проекта, разделенных на функциональные группы. Описаны архитектурные решения, стек технологий и бизнес-логика каждого компонента.

---

## 🏗️ 1. Глобальные дашборды и Системные интерфейсы

### `main_dashboard.html` (Оперативный дашборд рудника)
*   **Назначение**: Центральный узел мониторинга рудника "Инкай". Объединяет данные по транспорту, событиям ERC, сооружению скважин и ПАБ.
*   **Технические особенности**:
    *   Использование **CSS Variables** для управления темами и цветовыми кодами статусов (In Work, Reserve, Plan TO, Rem).
    *   Сложная сетка `grid-template-columns: repeat(auto-fit, minmax(320px, 1fr))` для адаптации под любые мониторы.
    *   Интеграция с системой `openModule` для бесшовной навигации между модулями платформы.
*   **Функционал**:
    *   Живые KPI-карточки со статус-барами.
    *   Вкладки по периодам (Сегодня/Месяц/Год) для динамических срезов данных.
    *   Инфографика по участкам и видам скважин.

### `admin-panel.html` (Универсальная административная панель)
*   **Назначение**: Мастер-шаблон для создания административных интерфейсов.
*   **Технические особенности**:
    *   Огромная база CSS-компонентов: кастомные чекбоксы, переключатели, стилизованные таблицы и формы.
    *   Система анимаций для модальных окон (`@keyframes modalFadeIn`) и всплывающих уведомлений.
    *   Реализация поиска-фильтра прямо в заголовке панели.
*   **Компоненты**: Статистические карточки с иконками, таблицы с пагинацией, формы редактирования с валидацией.

### `lab_main_menu.html` (Меню Лаборатории)
*   **Назначение**: Эталонный интерфейс навигации для лабораторных модулей (Добыча, Переработка, Экология).
*   **Дизайн**: Минимализм, использование градиентных иконок для навигационных кнопок и hover-эффектов с масштабированием (`transform: scale(1.02)`).

## 🛠️ 2. Специализированные производственные инструменты

### `learning_application.html` (Запись на обучение)
*   **Назначение**: Автоматизация процесса подачи заявок на обучение.
*   **Логика**: Класс `DataService` инкапсулирует все запросы к API (курсы, расписание, заявки).
*   **UX-фишки**: График "шахматка" для наглядного выбора времени. Автоматический расчет доступных мест.

### `qr_mesto.html` (Инвентаризация через QR)
*   **Назначение**: Учет и маркировка технологических точек.
*   **Технологии**: Библиотека `QRCode.js` для генерации кодов на стороне клиента.
*   **Особенности**: Специальный CSS-медиазапрос `@media print` для идеальной верстки наклеек на бумаге.

### `transport_profile.html` (Карточка транспортного средства)
*   **Назначение**: Детальное досье на единицу техники.
*   **Интеграция**: Вывод треков из системы Omnicomm на карту Leaflet.js.
*   **Функционал**: История статусов, технические характеристики, быстрый доступ к документации.

## 📈 3. Аналитика и Мониторинг (РВР и МРЦ)

### `mrc.html` (Дашборд Межремонтного цикла)
*   **Назначение**: Визуализация жизненного цикла скважин и оборудования.
*   **Уникальность**: Реализован сложный Zoom/Pan механизм для горизонтального таймлайна, позволяющий рассматривать данные как в масштабе года, так и конкретного дня.
*   **Визуализация**: Heatmap-сетка для распределения работ по горизонтам.

### `rvr_dashboard.html` (Таблица РВР)
*   **Назначение**: Глобальный реестр ремонтно-восстановительных работ.
*   **Технологии**: **DataTables.js** с плагинами FixedColumns и Buttons.
*   **Функционал**: Продвинутая фильтрация (Скважина, Блок, Метод РВР, Даты) и экспорт в Excel.

### `well_debit_dashboard.html` (Мониторинг дебитов)
*   **Назначение**: Контроль производительности урановых скважин.
*   **Визуализация**: Chart.js для отображения динамики (Line chart) и распределения по типам (Doughnut).

## 🐍 4. Автоматизация и Backend-утилиты (Python)

### `make_mrc.py`
*   **Роль**: Скрипт-генератор. Он берет HTML-прототип и автоматически преобразует его в рабочее приложение:
    *   Заменяет "демо-данные" на JS-функции `fetchData`.
    *   Внедряет обработку заголовков авторизации.
    *   Переписывает IIFE-блоки в именованные функции для возможности обновления графиков "на лету".

### `prois_control_df.py`
*   **Роль**: ETL-инструмент для Excel.
    *   Находит строку заголовка с неделями, используя умный поиск.
    *   Разворачивает таблицу из широкого формата (недели в колонках) в длинный (tidy data) для баз данных.
    *   Использует библиотеку `pandas`.

### `test_parser.py`
*   **Роль**: Интеллектуальный парсер Word-документов.
    *   Разбирает таблицы с вопросами.
    *   Использует `python-docx` для анализа стилей текста (поиск красного/зеленого цвета шрифта для определения правильных ответов).
